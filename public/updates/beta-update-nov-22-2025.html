üí¨ Coach Creator: From Goldfish Memory to Agentic Memory

The old flow had goldfish memory. You'd tell it "I train at a CrossFit gym with full equipment" in question 2, and it would ask about your equipment again in question 5. Frustrating. Robotic. Made you repeat yourself constantly.

The new flow has agentic memory. It remembers everything you've said across the entire conversation.

  • Persistent context tracking: Mention equipment once? It's remembered. Won't ask again.
  • Natural conversation flow: Questions adapt based on what you've already volunteered, not a rigid script
  • Semantic storage: Not just keyword matching ‚Äì actual understanding of what you meant
  • Richer coach intelligence: Your coach gets built from deeper context because the system remembers nuance
  • Adaptive pacing: Clear and direct? Finishes fast. Want to elaborate? It adapts.

We watched dozens of coach creation sessions. Every time someone said "I already told you that" ‚Äì that was goldfish memory. We replaced it with agentic memory. Problem solved.

üèãÔ∏è Workout Logging: Complex Workouts, Simple Process

The old extraction struggled with multi-phase workouts. Strength session followed by a metcon? It got confused about total duration. Image attached but AI only read text? Data loss. AMRAP with weird timing? Wrong calculations.

The new system handles real workouts.

  • Image-based logging: Snap a whiteboard photo. AI reads it. Done. Works with messy handwriting and CrossFit notation.
  • Multi-phase workouts: Back squat session + AMRAP finisher? AI correctly sums components and tracks both workout duration and total gym time.
  • Session vs workout duration: New session_duration field captures total gym time while duration tracks pure work time.
  • AMRAP duration fixed: 10-minute AMRAP now correctly shows 10 minutes, not some calculated average.
  • Better validation: AI gets the actual schema as reference when normalizing data ‚Äì fewer mistakes, better accuracy.

If you can log it on a whiteboard, we can extract it from an image. That's the goal. We're getting closer.

üèóÔ∏è The Architecture Story (This Is the Foundation)

We've always used multiple AI patterns (never just one giant AI call). But we introduced several new smart agentic patterns that dynamically activate based on what needs to be done:

Orchestrator/synthesizer patterns: Specialized agents now handle detection, extraction, validation, and storage independently. Each agent focuses on what it's best at.

Evaluator/optimizer patterns: Validate data quality before storage. If the AI makes a mistake, the evaluator catches it. If a field is missing, the optimizer infers it. Resilient by design.

Parallel processing: Router analysis + data loading + config fetching now happen simultaneously. 2-3x faster when it matters.

Tool-based generation: Using Bedrock's tool-use APIs lays the foundation for future MCP server integrations. This is how we'll connect to SugarWOD, Push Press, Apple Health, and other platforms. We're building the plumbing now so those integrations are fast later.

Runtime upgrade: All cloud functions upgraded to Node.js 22 runtime via centralized configuration. Better performance, improved security, and simplified runtime management across the entire backend infrastructure.
