# Long-form and multi-subject extensions

The default deliverable is **shareable summary + ~2–3 pages of story** in one doc. Use this file only when the user explicitly asks for **extra depth**, appendices, or multi-user synthesis.

## Extended sections (add after core template)

- **Comparative baseline**: how this subject differed from a typical onboard if cohort stats exist (otherwise omit).
- **Feature touchpoints**: table mapping observed actions to product surfaces only when logs justify it.
- **Appendix: event timeline**: bullet list of dated events for internal review; trim for public doc.
- **Metrics glossary**: definitions for any domain terms (RPE, volume, adherence) used in the paper.

## Multi-subject synthesis

- State clearly that the narrative synthesizes **multiple anonymized individuals**.
- Avoid implying one person achieved everything listed.
- Prefer ranges and patterns ("most users in this sample…") with n and caveats.
